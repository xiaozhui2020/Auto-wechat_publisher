

### ? 绿色竖线（例如第69行）

绿色线表示这一行是 **新增的** ，也就是说这行内容是你自上次 Git 提交以来新添加的内容，还没有被提交到 Git。

---

### ? 蓝色竖线（例如第77-78行）

蓝色线表示这一行是 **已修改的** ，即该行内容之前存在，但你做了改动。这些修改也还未提交到 Git。

---

### ? 红色竖线（如果有）

红色线（你的图中没有，但顺便讲一下）表示这一行是**被删除的**内容，在当前文件中已经看不到原内容了，但 VS Code 会在版本控制中标记这些删除。

---

### ?? 淡灰色文字（例如第59、62、65、71行）

这些是注释代码，前面带有 `#`。并不是 Git 的标记，只是 Python 的注释语法。

---

### 提示：如何查看和提交这些变动？

* **查看修改详情** ：可以点击 VS Code 左侧的 Source Control 图标（通常是一个分支图标），查看每个文件的具体改动。
* **提交修改（commit）** ：点左侧 Source Control 面板中的 “?” 图标，填写提交说明，即可完成提交。
